Design System
Tiny Wire
A minimal, warm-toned design system built for dense operational interfaces. Two fonts, a coherent color language, and a small set of reusable components.
Usage
Drop globals.css into any project. All tokens are CSS custom properties on :root.
/* 1. Import the token file */ @import './tokens/globals.css'; /* 2. Import fonts in your <head> */ /* Bricolage Grotesque + DM Sans from Google Fonts */ /* 3. Use tokens anywhere */ .my-card { background: var(--surface); border: 1px solid var(--border); border-radius: var(--radius-md); padding: var(--space-16); box-shadow: var(--shadow-card); }
/* JS / TS — import tokens object */ import tokens from './tokens/design-tokens.js' // Access any value const brand = tokens.colors.brand // '#2F4BA6' const base = tokens.typography.fontSize.base // 13

Buttons
Five variants covering the full action spectrum from primary confirmation to destructive hard-stop.
/* HTML */ <button class="btn-primary">Apply changes</button> <button class="btn-danger">Confirm delete</button> <button class="btn-stop">Stop all dialing</button> <button class="btn-rollback">Rollback</button> <button class="btn-ghost">Cancel</button>
Tags & Badges
Tags are loud: all-caps, colored fill. Badges are quiet: muted, used for metadata like "rolled back".
Hard stop Active Info Review rolled back v1.2

Inputs & Selects
32px tall, 6px radius. Full width by default; .input-sm for inline numeric fields (72px wide). Focus state swaps border to --text-tertiary.

Cards
The primary container unit. White surface, 1px border, 8px radius, 16px padding. Disabled state dims to 45% opacity.
